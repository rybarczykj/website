Why you should use bpython

08/15/2020

Intro

Anyone who's used the python interactive interpreter will tell you the UI sucks. It's truly a great tool for quickly typing python code, but if you're not perfect —flawless syntax and indentation, incredible forethought, ability to thrive in a boring black-and-white environment, and total omniscience of all method names and arguments— your terminal will quickly fill up with errors.

Enter bpython. A beautiful, wonderful, lightweight, free, open-source, interface to the python interpreter here to save the day. If you ask me, all python developers should use it. But don't take my word for it; I worked on the project. Try it yourself!

Interactive Demo

The REPL below already has bpython installed. Just type bpython to start it up. Note: browser performance is a little slow and some features are buggy, but you should be able to get an idea of the features.

Things to try

  • Play around a bit. You'll quickly notice syntax highlighting as you type. Helpful!
  • Make a mistake? Press control + R to undo your last input. (this feature alone was enough to convince me).
  • Try defining a class or function. Auto indentation, syntax highlighting, and undo should make it easy.
  • Type import math (or some other library) and then math. to see suggestions. Tab through them and type an open paren to see the method signature!

Convinced? Go download it! Just type pip install bpython from the command line and you're good to go. Then try these other features.

  • Press F7 to edit your bpython session in progress. Make some changes to inputs, and when you close the file, the session will recompile with your changes.
  • Don't like vim as the default editor? Me neither. Open the config file with F3, and go to line ~40 and follow the simple instructions to change it to VSCode or Atom. There are some other useful configurations in that file as well.

For more info, check out bpython's Github or website.
